2Qué datos se guardan en cada pedido

Cada vez que un cliente escanea el QR, arma su pedido y paga, se crean registros en estas tablas:

TablaDescripciónSe crea cuando…
ordersPedido principal: mesa, total, estado, método de pagoEl cliente confirma el pago
order_itemsCada plato dentro del pedido (nombre, precio, cantidad)Inmediatamente después del pedido
order_item_extrasExtras seleccionados por plato (panceta, huevo, etc.)Si el cliente eligió extras
order_status_historyLog de todos los cambios de estadoCada vez que cambia el estado
ratingsCalificación 1–5 estrellas + comentarioAl final del flujo (opcional)
waiter_callsLlamadas al mozo desde la appSi el cliente presiona "Llamar mozo"

Flujo de estados de un pedido

draft → confirmed → paid → kitchen_received → cooking → ready → delivered
Para las estadísticas Filtrar siempre con status NOT IN ('draft', 'confirmed') para excluir pedidos que nunca se completaron. Los estados resaltados en verde son los que maneja la cocina.

3Ver datos visualmente (Table Editor)

Ver todos los pedidos

  1. Clic en Table Editor en la barra lateral izquierda
  2. Seleccionar la tabla orders
  3. Ver las filas y columnas directamente en pantalla

Filtrar por fecha

  1. En el Table Editor, hacer clic en el botón Filter
  2. Seleccionar columna created_at
  3. Elegir condición is after e ingresar la fecha
  4. Clic en Apply

Ver los ítems de un pedido específico

  1. Copiar el id (UUID) del pedido que te interesa
  2. Ir a la tabla order_items
  3. Filtrar por order_id = el UUID copiado

4Consultas SQL para estadísticas

Ir a SQL Editor → clic en New query → pegar la consulta → clic en Run (o Ctrl+Enter).

4.1 — Pedidos de hoy

pedidos_hoy.sqlSQL
SELECT
  o.order_number,
  t.number          AS mesa,
  o.status,
  o.total,
  o.payment_method,
  o.created_at
FROM orders o
LEFT JOIN tables t ON t.id = o.table_id
WHERE DATE(o.created_at) = CURRENT_DATE
  AND o.status != 'draft'
ORDER BY o.created_at DESC;

4.2 — Resumen del día (ingresos y métodos de pago)

resumen_dia.sqlSQL
SELECT
  COUNT(*)                          AS total_pedidos,
  SUM(total)                        AS ingresos_totales,
  ROUND(AVG(total))                 AS ticket_promedio,
  SUM(discount_amount)              AS descuentos_aplicados,
  COUNT(*) FILTER (WHERE payment_method = 'efectivo') AS pagos_efectivo,
  COUNT(*) FILTER (WHERE payment_method = 'tarjeta')  AS pagos_tarjeta,
  COUNT(*) FILTER (WHERE payment_method = 'qr')       AS pagos_qr
FROM orders
WHERE DATE(created_at) = CURRENT_DATE
  AND status NOT IN ('draft', 'confirmed');

4.3 — Platos más vendidos (todos los tiempos)

top_platos.sqlSQL
SELECT
  oi.item_name,
  SUM(oi.quantity)                   AS unidades_vendidas,
  SUM(oi.quantity * oi.unit_price)   AS ingresos_generados
FROM order_items oi
JOIN orders o ON o.id = oi.order_id
WHERE o.status NOT IN ('draft', 'confirmed')
GROUP BY oi.item_name
ORDER BY unidades_vendidas DESC
LIMIT 10;

4.4 — Platos más vendidos (esta semana)

top_platos_semana.sqlSQL
SELECT
  oi.item_name,
  SUM(oi.quantity)                     AS unidades,
  SUM(oi.quantity * oi.unit_price)     AS ingresos
FROM order_items oi
JOIN orders o ON o.id = oi.order_id
WHERE o.status NOT IN ('draft', 'confirmed')
  AND o.created_at >= date_trunc('week', NOW())
GROUP BY oi.item_name
ORDER BY unidades DESC
LIMIT 10;

4.5 — Ingresos por día (últimos 30 días)

ingresos_diarios.sqlSQL
SELECT
  DATE(created_at)  AS fecha,
  COUNT(*)          AS pedidos,
  SUM(total)        AS ingresos
FROM orders
WHERE status NOT IN ('draft', 'confirmed')
  AND created_at >= NOW() - INTERVAL '30 days'
GROUP BY DATE(created_at)
ORDER BY fecha DESC;

4.6 — Ingresos por semana (últimas 8 semanas)

ingresos_semanales.sqlSQL
SELECT
  DATE_TRUNC('week', created_at) AS semana,
  COUNT(*)                       AS pedidos,
  SUM(total)                     AS ingresos
FROM orders
WHERE status NOT IN ('draft', 'confirmed')
  AND created_at >= NOW() - INTERVAL '8 weeks'
GROUP BY DATE_TRUNC('week', created_at)
ORDER BY semana DESC;

4.7 — Mesa más activa

mesas_activas.sqlSQL
SELECT
  t.number     AS mesa,
  COUNT(*)     AS pedidos,
  SUM(o.total) AS ingresos
FROM orders o
LEFT JOIN tables t ON t.id = o.table_id
WHERE o.status NOT IN ('draft', 'confirmed')
GROUP BY t.number
ORDER BY pedidos DESC;

4.8 — Uso de cupones

cupones.sqlSQL
SELECT
  coupon_code,
  COUNT(*)              AS veces_usado,
  SUM(discount_amount)  AS ahorro_total_clientes
FROM orders
WHERE coupon_code IS NOT NULL
  AND status NOT IN ('draft', 'confirmed')
GROUP BY coupon_code
ORDER BY veces_usado DESC;

4.9 — Calificaciones de clientes

calificaciones.sqlSQL
SELECT
  ROUND(AVG(stars), 1)               AS promedio,
  COUNT(*)                           AS total_calificaciones,
  COUNT(*) FILTER (WHERE stars = 5)  AS cinco_estrellas,
  COUNT(*) FILTER (WHERE stars = 4)  AS cuatro_estrellas,
  COUNT(*) FILTER (WHERE stars = 3)  AS tres_estrellas,
  COUNT(*) FILTER (WHERE stars <= 2) AS una_dos_estrellas
FROM ratings;

4.10 — Tiempo promedio de preparación (de pago a listo)

tiempo_preparacion.sqlSQL
SELECT
  ROUND(
    AVG(
      EXTRACT(EPOCH FROM (ready_at - paid_at)) / 60
    )
  ) AS minutos_promedio_preparacion
FROM (
  SELECT
    o.id,
    MIN(h.changed_at) FILTER (WHERE h.status = 'paid')  AS paid_at,
    MIN(h.changed_at) FILTER (WHERE h.status = 'ready') AS ready_at
  FROM orders o
  JOIN order_status_history h ON h.order_id = o.id
  GROUP BY o.id
) t
WHERE paid_at IS NOT NULL AND ready_at IS NOT NULL;

4.11 — Extras más pedidos

top_extras.sqlSQL
SELECT
  oie.extra_name,
  COUNT(*)        AS veces_pedido,
  SUM(oie.extra_price * oi.quantity) AS ingresos_extra
FROM order_item_extras oie
JOIN order_items oi ON oi.id = oie.order_item_id
JOIN orders o ON o.id = oi.order_id
WHERE o.status NOT IN ('draft', 'confirmed')
GROUP BY oie.extra_name
ORDER BY veces_pedido DESC;

4.12 — Vista completa de ticket (pedido + ítems)

ticket_completo.sqlSQL
SELECT
  o.order_number,
  t.number     AS mesa,
  o.created_at,
  oi.item_name,
  oi.quantity,
  oi.unit_price,
  oi.quantity * oi.unit_price AS subtotal_item
FROM orders o
JOIN order_items oi ON oi.order_id = o.id
LEFT JOIN tables t ON t.id = o.table_id
WHERE o.status NOT IN ('draft', 'confirmed')
ORDER BY o.created_at DESC, oi.item_name;

5Ver actividad en tiempo real

Supabase tiene Realtime habilitado para estas tablas:

TablaQué evento escucha
ordersCambios de estado (cocina avanzando tickets)
order_status_historyCada registro nuevo de cambio de estado
waiter_callsLlamadas al mozo y cambios a "atendido"

Ver pedidos entrantes en vivo desde el dashboard

  1. Ir a Table Editor → tabla orders
  2. Filtrar por status = paid
  3. Refrescar la página para ver los nuevos pedidos
Tip El SQL Editor no se actualiza automáticamente. Para monitoreo en vivo, usar el Table Editor con filtros aplicados y refrescar periódicamente.

6Exportar datos a Excel / CSV

Desde el SQL Editor (recomendado para reportes)

  1. Correr cualquier consulta SQL
  2. Ver los resultados en la tabla inferior
  3. Hacer clic en el botón Export (esquina superior derecha de los resultados)
  4. Elegir CSV — el archivo se descarga directamente

Desde el Table Editor (para tablas completas)

  1. Abrir la tabla que querés exportar
  2. Hacer clic en el botón ··· (tres puntos, arriba a la derecha)
  3. Seleccionar Export to CSV
Los valores de total, subtotal y discount_amount están en guaraníes enteros (sin decimales). Al abrir en Excel, no dividir ni reformatear esos valores.
